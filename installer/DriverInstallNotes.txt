MacroHID Driver Installation Notes

MacroHID can be installed without the test driver. In that mode MacroStudio, MacroRunner dry-run, LatencyProbe, samples, and documentation are available, but real virtual HID keyboard/mouse submission is disabled.

Installing the MacroHID test driver enables:
- Virtual HID keyboard, mouse, and consumer-control reports.
- MacroRunner --send end-to-end tests against the local driver.
- Driver statistics such as submitted/rejected report counts.
- The low-latency path needed for real desktop automation measurements.

Development driver requirements:
- Administrator rights.
- Windows test-signing mode enabled.
- Secure Boot disabled if Windows blocks test-signing mode.

The installer can optionally run the driver install script after copying files. If Windows reports that test-signing is protected by Secure Boot policy, disable Secure Boot in UEFI/BIOS, boot Windows again, and rerun the driver install shortcut from the Start menu.
